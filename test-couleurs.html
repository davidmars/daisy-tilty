Couleurs de marque

Les couleurs identitaires du projet : primary, secondary, accent et neutral.

Couleur principale. Boutons, liens, éléments actifs.

primary bg-primary / text-primary

Couleur secondaire. Tags, accents complémentaires.

secondary bg-secondary / text-secondary

Accent vif. Mise en valeur ponctuelle, badges, highlights.

accent bg-accent / text-accent

Neutre profond. Barres de navigation, footers.

neutral bg-neutral / text-neutral

Couleurs d'état

Réservées aux retours système : information, succès, avertissement, erreur.

Information neutre. Notifications, bulles d'aide.

info bg-info / text-info

Confirmation d'action réussie. Formulaires validés.

success bg-success / text-success

Avertissement non bloquant. Alertes de précaution.

warning bg-warning / text-warning

Erreur bloquante. Validation échouée, états critiques.

error bg-error / text-error

Couleurs en contexte

Exemples concrets d'application des couleurs via les composants DaisyUI.

Boutons — .btn

Badges — .badge

primary secondary accent neutral info success warning error

Alertes — .alert

Information : la configuration a bien été chargée.
Succès : le formulaire a été envoyé avec succès.
Avertissement : certaines données sont incomplètes.
Erreur : la connexion au serveur a échoué.

Textes colorés — text-{couleur}

text-primary — Couleur principale de l'interface.

text-secondary — Couleur secondaire, accents complémentaires.

text-accent — Accent vif pour les mises en valeur.

text-neutral — Teinte neutre profonde.

text-info — Information neutre.

text-success — État de succès.

text-warning — État d'avertissement.

text-error — État d'erreur.
